# QA Release Checklist

Use this checklist before publishing release notes.

## Product Review

- [ ] Release version confirmed
- [ ] Release date confirmed
- [ ] Product owner reviewed release summary
- [ ] Feature list confirmed
- [ ] Fixed issues confirmed
- [ ] Known issues confirmed
- [ ] Breaking changes confirmed
- [ ] Upgrade notes confirmed

## QA Review

- [ ] Test status reviewed
- [ ] Failed tests documented
- [ ] Known issues include workarounds
- [ ] Severity levels confirmed
- [ ] Regression testing complete
- [ ] Screenshots or logs reviewed, if needed

## Documentation Review

- [ ] User-facing notes written in plain language
- [ ] Internal notes include engineering detail
- [ ] Changelog updated
- [ ] Version history updated
- [ ] Support notes included
- [ ] API changes documented
- [ ] Links checked
- [ ] Final proofreading complete
